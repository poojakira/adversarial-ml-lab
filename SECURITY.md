# Security Policy

## Reporting a Vulnerability
Report vulnerabilities privately to poojakiranbhardwaj@gmail.com. Do not open a public issue for exploitable findings.

## Supported Versions
Security fixes target the `main` branch until tagged releases are published.

## Verification
Before reporting a suspected issue, include the command, input, expected result, and observed result. For CI report integrity, set `ADV_LAB_HMAC_KEY` before running the benchmark harness so JSON output includes an HMAC-SHA256 signature.
